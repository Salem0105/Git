Comandos GIT

Uso de comandos git y adicionalmente una pequeña práctica de HTML y CSS
  • git init

    Creamos un repositorio de la carpeta actual, se puede crear una carpeta colocando un nombre después del init

  • git status

    Muestra el estado de los archivos o directorios, en rojo si solo se encuentran en el working directory y verde si está en el Staging area

  • git add < file >

    Agrega el archivo al Staging area, si ponemos un punto agregamos todo lo que esté en la carpeta o working directory

  • git rm --cached < file >

    Eliminamos el archivo del Staging area, para no hacerle commit, el archivo seguirá en el working directory

  • git commit -m "message"

    Mandamos lo que esté en el Staging area al git repository

  • git config --list

    Muestra la configuración de git

  • git config --list --show-origin

    Muestra donde están las configuraciones guardadas

  • git config --global user.name "NombreUser"

    Se establece el nombre del usuario

  • git config --global user.email "user@email.com"

    Se establece el correo del usuario

  • git show"

    Muestra el historial de cambios

  • git log"

    Muestra el historial de registros con detalles
